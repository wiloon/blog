  rem CODE BY t0nsha<br /> rem 关于提取date,time输出结果的一个批处理<br /> rem “:”（冒号）和“~”波浪号必不可少！<br /> rem “~”后的数字：为正数表示舍弃输出结果的前几位；直接跟负数表示取到输出结果的后第几位。<br /> rem “,”后的数字：为正数表示取到输出结果的前第几位；为负数表示舍弃输出结果的后几位。<br /> echo %date%<br /> echo %date:~4%<br /> ::下行表示舍弃前0位，取到第10位（即取输出结果的前10位）<br /> echo %date:~0,10%<br /> echo %date:~4,-5%<br /> pause<br /> echo %time%<br /> echo %time:~-3%<br /> echo %time:~2,-3%<br /> pause<br /> echo %date:~4% %time:~0,-3%<br /> pause</p> 
  
  <p>
    BTW
  </p>
  
  <p>
    使用批处理产生日期（时间）文件、文件夹 帮别人整Sql     server自动备份<br /> 发现无法使用网络映射驱动器作为备份文件存放路径<br /> 而本机磁盘空间实在是不够<br /> 于是决定在本机只备份最新2天数据<br /> 再写个批处理，做成系统调度<br /> 每周将备份数据复制到网络驱动器上存档
  </p>
  
  <p>
    从网上搜到批处理产生日期文件的办法<br /> 下面是实现的比较好的
  </p>
  
  <p>
    批处理文件：<br /> @echo off<br /> set aFile=bak-%DATE:~4,4%%DATE:~9,2%%DATE:~12,2%<br /> set bFile=bak-%TIME:~0,2%%TIME:~3,2%%TIME:~6,2%<br /> set cFile=bak-%DATE%<br /> echo Afile=%aFile%<br /> echo Bfile=%bFile%<br /> echo Cfile=%cFile%
  </p>
  
  <p>
    输出:<br /> Afile=bak-20061219<br /> Bfile=bak-113202<br /> Cfile=bak-星期二 2006-12-19
  </p>
  
  <p>
    于是备份bat就好写了<br /> @echo off<br /> echo 正在备份数据到网络驱动器。。。<br /> set folder=%DATE%<br /> md "y:%folder%&#8221;<br /> copy d:DataBak*.BAK "y:%folder%&#8221;<br /> echo 备份完毕。
  </p>
  
  <p>
    &#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;<br /> @echo off<br /> set AFile=bak-%DATE:~4,4%%DATE:~9,2%%DATE:~12,2%<br /> set BFile=bak-%TIME:~0,2%%TIME:~3,2%%TIME:~6,2%<br /> echo AFile=%AFile%.rar<br /> echo BFile=%BFile%.rar
  </p>
  
  <p>
    运行此批处理的结果：<br /> AFile=bak-20060109.rar &#8212;&#8212;- 年月日 &#8212; 8位<br /> BFile=bak-140650.rar &#8212;&#8212;&#8212;- 时分秒 &#8212; 6位
  </p>
  
  <p>
    另：如果小时数只有一位数字，造成中间有空格而出错的问题，请使用如下方法补0<br /> set hh=%time:~0,2%<br /> if /i %hh% LSS 10 (set hh=0%time:~1,1%)
  </p>
