## wifi

Wi-Fi是一种允许电子设备连接到一个无线局域网 (WLAN）的技术，通常使用2.4G UHF或5G SHF ISM 射频频段。

### 802.11a
5 GHz

1999 年发布。
各种调制类型的数据传输率：6、9、12、18、24、36、48 和 54 Mbps。
带 52 个子载波频道的正交频分复用 (OFDM）技术。
不需要许可证的国家信息基础设施 (UNII）频道内的 12 个 5 GHz 互不重叠频带。

### 802.11b
2.4 GHz
传送速度为11Mbit/s
IEEE802.11b的后继标准是IEEE802.11g；
在2.4-GHz-ISM频段共有14个频宽为22MHz的频道可供使用。

1999 年发布。
各种调制类型的数据传输率：1、2、5.5 和 11 Mbps
高速直接序列展频 (HR-DSSS）。
工业、科技和医疗 (ISM）领域内的三个 2.4 GHz 互不重叠频带。

### 802.11g
2.4GHz OFDM
原始传送速度为54Mbit/s，净传输速度约为24.7Mbit/s (跟802.11a相同）。

2003 年发布。
各种调制类型的数据传输率：6、9、12、18、24、36、48 和 54 Mbps；可以使用 DSSS 和 CCK 转换为 1、2、5.5 和 11 Mbps。
带 52 个子载波频道的正交频分复用 (OFDM）技术；使用 DSSS 和 CCK 向下兼容 802.11b。
工业、科技和医疗 (ISM）领域内的三个 2.4 GHz 互不重叠频带。

### 802.11n, WiFi 4
802.11n是在802.11g和802.11a之上发展起来的一项技术，最大的特点是速率提升，理论速率最高可达600Mbps (目前业界主流为300Mbps）。

802.11n可工作在2.4GHz和5GHz两个频段

各种调制类型的数据传输率：1、2、5.5、6、9、11、12、18、24、36、48 和 54 Mbps (参见下表）
采用多输入多输出 (MIMO）和频道绑定 (CB）的正交频分复用 (OFDM）技术
工业、科技和医疗 (ISM）领域内的三个 2.4 GHz 互不重叠频带
无论有无 CB，均为不需要许可证的国家信息基础设施 (UNII）频道内的 12 个 5 GHz 互不重叠频带
注
我们建议 5 GHz 的通道连接，因为 2.4 GHz 波段中有数量有限的非重叠通道。


### 802.11ac协议  (WiFi 5） 5GHz MU-MIMO
IEEE 802.11ac，是一个802.11无线局域网 (WLAN）通信标准，它通过5GHz频带 (也是其得名原因）进行通信。理论上，它能够提供最多1Gbps带宽进行多站式无线局域网通信，或是最少500Mbps的单一连接传输带宽。802.11ac是802.11n的继承者。

### 80211  a n ac
802.11ac简介
IEEE 802.11ac,是一个802.11无线局域网 (WLAN) 通信标准,它通过5GHz频带 (也是其得名原因) 进行通信。理论上,它能够提供最多1Gbps带宽进行多站式无线局域网通信,或是最少500Mbps的单一连接传输带宽。



802.11ac是802.11n的继承者。它采用并扩展了源自802.11n的空中接口 (air interface) 概念,包括: 更宽的RF带宽 (提升至160MHz) ,更多的MIMO空间流 (spaTIal streams)  (增加到 8) ,多用户的 MIMO,以及更高阶的调制 (modulaTIon)  (达到 256QAM) 。

## 无线安全模式

路由器里的 WPA2-PSK、WPA3-SAE 等是**无线加密（认证）方式**，与上文 802.11a/b/g/n 等**物理层协议代际**不是同一维度。家用场景通常用**同一个 Wi-Fi 密码**连网；WPA3 的 SAE 在体验上仍像输密码，但握手机制更安全。

### 常见四种模式

| 模式 | 说明 | 优点 | 缺点 |
| ---- | ---- | ---- | ---- |
| WPA2-PSK | 仅 WPA2，密码 + AES 加密 | 成熟、兼容最好 | 弱密码存在被离线穷举的风险 |
| WPA/WPA2-PSK | 混合：同一 SSID 同时兼容 WPA 与 WPA2 客户端 | 极老设备（只支持 WPA）也能连 | 整体安全性被 WPA 拉低；现已少见 |
| WPA3-SAE | 仅 WPA3，使用 SAE 握手 | 抗弱密码、抗部分握手攻击更好 | 很老的终端可能连不上 |
| WPA2-PSK/WPA3-SAE | 过渡混合：新设备走 WPA3，老设备仍可用 WPA2 | 新旧设备混用时较省心 | 安全性介于纯 WPA3 与纯 WPA2 之间 |

- **PSK**（Pre-Shared Key）：预共享密钥，即常见的 Wi-Fi 密码。
- **SAE**（Simultaneous Authentication of Equals）：WPA3 的认证方式，密码输入体验类似，协议层更安全。

### 家用路由器上哪种最常见

近十年出货的路由器，默认多为 **WPA2-PSK** 或 **WPA2/WPA3 过渡（WPA2-PSK/WPA3-SAE）**。**WPA/WPA2-PSK** 多见于多年前的设备或特意为古董机开的兼容模式，现已不是主流默认。全家设备都较新时，可单独使用 **WPA3-SAE**。

加密算法请选 **AES**，避免 TKIP。

### Nintendo Switch 2 支持情况

[任天堂官方说明](https://en-americas-support.nintendo.com/app/answers/detail/a_id/68530/)：Switch 2 支持 **WPA-PSK (AES)**、**WPA2-PSK (AES)**、**WPA3-SAE (AES)**，**不支持 WEP**。不支持的加密类型会提示网络设置不受支持。

| 路由器选项 | Switch 2 |
| ---------- | -------- |
| WPA2-PSK | 明确支持，兼容性最好 |
| WPA/WPA2-PSK | 一般可用（实际连 WPA2 档） |
| WPA3-SAE | 明确支持 |
| WPA2-PSK/WPA3-SAE | 通常可用（按设备能力选 WPA2 或 WPA3） |

**给 Switch 2 的实用建议：**

1. 优先使用 **WPA2-PSK (AES)**，连不上时再排查信道、5 GHz、隐藏 SSID 等。
2. 全家设备较新且路由器稳定时，可尝试 **WPA3-SAE** 或 **WPA2/WPA3 过渡**。
3. 除非仍有只支持 WPA 的老设备，否则不必为 Switch 2 单独开 **WPA/WPA2-PSK**。
