早上跑步回来的路上， 我在思考一个问题， 裁员之后，暂时没有理想的 offer, 除了继续找适合的职位,投简历，我是不是应该趁这个机会测试 一下我自己加上 ai 能不能创造一些价值（赚到钱），然后回到家之后 我在 ticktick（一个 todo list） 里创建了一个 "倒数日" 想是能直观的看到我距离裁员的 last day有多少天了， 又因为 ticktick的 倒数日要取个名字，直觉是写“失业” 但是感觉 这个词有点偏负面，所以想选一个中性的词，后来 选了 Exploration

 
**留下可回看的记录**：把想法、决定和进展写下来。**目标是记录，不是共鸣**——若有人读后觉得有共鸣，可以是结果，但不作为写作动机；同样不是诉苦帖，也不写成「可复制攻略」。

## 语气与风格

| 维度 | 要求 |
| ---- | ---- |
| 情绪浓度 | **低**；偏 **观察** 角度叙事，少用内心感受词 |
| 语气 | **陈述事实**；**客观叙事**，不说教；不用提醒、劝诫读者的口吻（如「你要…」「你应该…」） |
| 重心 | **就事论事**——陈述事实、决定、进展；讨论 **信息/标签编码什么**（如「失去什么 / 要做什么」），而非指向「你」该做什么 |
| 读者感受 | 作者在 **叙述一件事**（发生了什么、为什么这样想、做了什么决定），而非宣泄情绪或向读者喊话 |

## 格式

| 维度 | 要求 |
| ---- | ---- |
| 加粗 | **默认不加粗**；不为「方便扫读」给日期、主题词、实验问题、数字对比等加粗 |
| 小标题 | 分段用 `###`，不用 `**...**` 代替标题 |
| 例外 | 仅 Spec 明确允许的术语对比可保留加粗（本篇：**失去什么** / **要做什么**） |

**示例**：

- TickTick 取名：「但是看起来有点奇怪」✅ / 「但越看越别扭」❌——语义判断 vs 情绪披露
- 标签对比：「不描述 **要做什么**」✅ / 「不描述 **你要做什么**」❌——信息编码 vs 劝诫读者
- 加粗：全篇仅「失去什么 / 要做什么」一处 ✅ / 日期、三问、几小时·几秒等处处加粗 ❌

我上一份工作在一家外企，公司近期也裁员。last day 是 **2026 年 4 月 24 日**。公司给了 **数月赔偿**，算是接下来一段时间的生活缓冲。

裁员之后，我打算回归/或者继续 **远程工作**，四年前就想找一份远程工作，但是当时有朋友推荐,然后进了那家外企；公司推行混合办公，每周有n天可以在家远程，也算一定程度体验过远程工作。

早上跑完步，在回家的路上，脑子里一直在转同一个问题：

> 裁员之后，下一个理想的 offer 还没落定——我应不应该趁这段缓冲期，**测试一下**：我自己加上 AI，能不能 **创造出一些价值**（赚到钱）？

或许不必默认回到「找工作 / 打工」的状态；

回到家，我打开 TickTick，新建了一个 **倒数日**——从 last day 起，记录这段日子过去了多少天。

取名时，直觉想填 **「失业」**——字面上没错，指的就是「裁员到下一个 offer 之间」这段日子；但是看起来有点奇怪：它只描述 **失去什么**，不描述 **要做什么**。

每天打开 TickTick，能看到日期的累加

## 实验要回答什么问题

1. **我和 AI 能不能在真实场景里创造价值？**（不只是写 demo，而是有人愿意用、能省时间、能复用。）
2. **这种价值能不能部分转化成收入？**（工具、咨询、小产品、开源项目的可持续性等。）
3. **若答案偏 yes，是否足够支撑「自运营」？**

如果几个月后数据不好看，再去找工作。

## 已经有的第一个样本

就在想清楚「要不要做实验」的这几天，已经有一个很具体的小成果。

家人从事翻译相关工作，交付给客户的译文往往要做大量 **格式与规范校验**——过去同类文档要花 **几个小时** 人工核对。我和 AI 花了几个小时做了一个校验工具，同样的检查现在 **几秒钟** 就能跑完，而且对同一客户的后续项目可以 **重复使用**。

这事说明：

- 几小时级别的投入，可以节省别人未来 N 个小时的时间
- 场景是真实的、可交付的，不是玩具；
- AI 在这里是 **搭档**，负责生成和迭代，我负责需求、验收和边界。

这是这段 **探索** 里第一个可以写进笔记的 **证据点**。

## 接下来的方向

缓冲期有限，我会按「可演示、可复用、能记账」来排优先级，而不是散点学技术。

**贴近真实需求、容易形成复用**

- 继续帮我老婆打磨类似「几小时 → 几秒」的工具：规则清晰、重复劳动多的环节（校验、一致性检查、格式处理等）。我暂时没有 offer, 但是我可以用我的代码把她先解放出来。

**自己的长期项目**

- 开源：**ENX** 等——在 runway 内推进到能讲清「对谁有什么用」。
- 若干私有仓库里的想法——验证能否形成稳定用法，或小规模的"创造价值"

**每周复盘**

- 这周创造了什么？（钱、省下的时间、可演示的功能、开源进展都算，但要写实。）
- 哪件事离「可重复」最近？
- 下周只押注一件事是什么？

## 和「找工作」之间的关系

若这几个月能证明「自己 + AI」可以持续交付价值，甚至覆盖一部分生活成本，就多一条值得走的路线；若证明不了，履历和作品集也会更清晰，回去职场同样是加分项。

裁员可以理解成 "打击"，它也可以是一个 **转型窗口**。窗口不会很长，但足够试一次。
