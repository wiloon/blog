# SDD 交付物共通文风

> **适用范围**：仅 **有对应 `.ai/specs/{slug}.md` 的 SDD 交付物**（`content/post/.../{slug}.md`）。  
> **不适用于**：无 Spec 的普通改稿、转载/remix 老文、技术笔记等——那些只遵守 [AGENTS.md](../AGENTS.md) 的 Markdown 与 front matter 规则，**不受**本文「默认不加粗」等 prose 约束。

各篇 Spec 的 **事实、结构、禁区** 仍以单篇为准；本文约定 **SDD 跨篇共通** 的正文 prose 与 Markdown 用法，避免在每个 Spec 里重复写「默认不加粗」等。

## 文档层级（冲突时谁优先）

| 层级 | 文件 | 管什么 |
| ---- | ---- | ------ |
| 1 | [AGENTS.md](../AGENTS.md) | 全站改文底线：front matter、`url`、`lastmod`、标签、MD025、表格/代码块、中英文空格等 |
| 2 | **本文（delivery-style.md）** | SDD **交付物正文** 的默认语气与格式（加粗、标题层级等） |
| 3 | `.ai/specs/{slug}.md` | 本篇主题、必写/禁写、结构；**仅在本篇写明「例外」时**可覆盖本文默认 |

单篇 Spec 的 §语气与风格 / §格式 应写 **本篇特有** 内容，并注明「其余见 [delivery-style.md](delivery-style.md)」。

## 格式（默认）

| 维度 | 要求 |
| ---- | ---- |
| 加粗 | **默认不加粗**（`**...**`）；不为扫读给日期、Flight 编号、主题词、清单项、数字对比等加粗 |
| 小标题 | 用 `##` / `###` 分段；**不用** `**...**` 充当标题（如 `**事实：……**` 宜改为 `###` 或普通句） |
| 列表 | 无序列表用 `-`；列表与代码块前后空行 |
| 链接 | 引用外部事实时优先链到一手来源（官方页、本人 blog 等） |

### 加粗例外（仅当单篇 Spec 写明）

- 在 Spec §格式 的 **「本篇例外」** 行列出允许加粗的短语（如 exploration 的「失去什么 / 要做什么」）。
- 未列出的条目 **一律不加粗**。
- 科普/技术篇 **默认不** 因 Flight 编号、术语英文而加粗；术语首次出现用正文叙述即可。

## 语气（默认）

| 维度 | 要求 |
| ---- | ---- |
| 语气 | 陈述事实；不说教；不用「你要…」「你应该…」 |
| 情绪 | 偏观察、低情绪浓度；少夸张词 |
| 立场 | 探索/笔记可承认不确定；不包装成权威教程或终极结论 |

叙事类、科普类、个人记录的 **具体重心**（如 exploration 的 TickTick 标签、booster 的「测试轨道猜测」）写在 **单篇 Spec §语气与风格**，不要写进本文。

## 人称（默认：适度用「我」）

**淡化 ≠ 不用。** 探索类 SDD 交付物要有个人笔记感：该用「我」的地方要保留，不该用的地方不堆砌。

| 维度 | 要求 |
| ---- | ---- |
| 宜用「我」 | 疑问从哪来（看完发射后的 **我的** 疑问）、**我** 当时的猜测、对照材料后的转折（「从 …… 上看，……」可接 **我** 整理的理解）、回应猜测的收束（如 §误区 澄清 **我** 的某个念头）、**写在最后**（成文过程、Gemini、探索记录） |
| 宜用客观表述 | 技术机制、时间线、Flight 编号与公开事实：优先「从 …… 上看」「公开记录里」、无主语句 |
| 不宜 | 每段开头「我觉得」、技术段落反复「我认为」、「我看过……之后」式啰嗦（改用「从 …… 上看」） |
| 科普/技术主体 | 以客观叙述为主；**不必**把「我」清零，但同一段内「我」不宜超过 1 次 |
| 全文例外 | exploration 等个人叙事：Spec 写「人称例外：正文可更密集使用第一人称」 |

**示例**：

- ✅ 开篇：「我有一个疑问……我一度猜是否与测试轨道有关」+ 次段：「从 SpaceX 公布的 …… 上看，……下面是我按时间线整理的理解」
- ✅ 误区段末：「这也说明了我关于测试轨道的猜测不成立」
- ❌ 机制段：「我觉得热分离时一级速度约 6000 km/h……」
- 「我看过……之后」→「从 …… 上看」（材料作主语，必要时后接「我整理如下」）

## AI 改稿时

1. 读单篇 Spec → **本文** → AGENTS.md → 再改正文。
2. 成稿后自查：正文加粗是否 **仅** 出现在 Spec 允许的例外列表中；若过多，删减加粗而非新增。
3. 自查人称：技术段是否堆砌「我」；疑问、猜测、探索段是否 **过少** 以致不像个人笔记——必要处应保留「我」。
